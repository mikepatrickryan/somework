Exhibit 3.0

Publishing Framework for Large Scale Data-Rich Interactive Web Pages

Exhibit 3.0 is a publishing framework for large scale data-rich interactive Web pages, building on the success of the original Exhibit.

The Exhibit 3.0 software is comprised of two separate components, Scripted for building in-browser Exhibits, and Staged for server-based Exhibits. We are currently at a beta release for both pieces: most things work, but there are bugs and the packaging is a little rough around the edges.

Staged

Our main implementation focus for Exhibit 3.0 Staged is Backstage, on GitHub. Bug reports belong on GitHub. You will need to build it yourself to test it out, and here is the documentation on how to do that. You'll also find documentation on the new HTTP interface.

Scripted

Exhibit 3.0 Scripted is on GitHub. Bug reports belong on GitHub. You can see some demos on the continuous integrator, along with documentation and code coverage reports.

Beta Release!

Exhibit 3.0 Beta is ready for community input. Highlighted features included in this release:

  • Scalability: Browse 100k items
  • Persistence: Pick up where you left off browsing an Exhibit
  • Bookmarks: Share exactly what you see with others
  • jQuery: All Scripted code makes use of jQuery, resolving old browser-specific bugs
  • Result paging, bug fixes, documentation, tests, and more

What's still left:

  • Adding more views to both Scripted and Staged
  • Adding more facets to both Scripted and Staged
  • Adding more importer formats to Scripted
  • Fixing known bugs and adding back prior features

Zepheira will provide a hosted server environment in the future for testing the new Exhibit platform in the near future. In the meantime, you can build it yourself.

Background

Development work proceeds from the proposed architecture (PDF) plan released in early 2011. You can also read the official press release (PDF) for background information and the workshop summary (PDF) that originally defined new community requirements for Exhibit 3.0, including:

  • Be scalable up to 1 million triples and 20,000 facet values
  • Provide a server-side option for publishing large collections
  • Include a simple way to move from small collections stored in the browser to larger collections stored on the server
  • Support incremental updates of views in the UI
  • Provide an API for result set pagination and/or summarization
  • Make it easier to add new views, widgets and facets to the UI
  • Improve embed-ability of Exhibits into external applications and Web pages
  • Support links to particular Exhibit states
  • Provide support for limited data and UI editing in the Exhibit UI via plug-ins or widgets
  • Provide data export features to enable large-scale data editing in external applications (e.g. Google Refine)
  • For complex UI layout editing, provide extension hooks and reference implementations as examples
